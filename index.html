从零开始部署指南
完成后你将拥有一个永久可访问的链接，可以直接发到微信群。全程约 15 分钟。
1
获取 DeepSeek API Key
1. 打开 platform.deepseek.com，点击右上角「注册」
2. 用手机号或邮箱注册，完成验证
3. 登录后点击左侧「API Keys」→「创建 API Key」
4. 复制生成的 Key（格式为 sk-xxxxxxxx）
5. 妥善保存，页面关闭后无法再次查看

DeepSeek 新注册用户有免费额度，分析普通研究报告成本极低（约 ¥0.01/篇）。
💡 将 API Key 填入本工具的「设置」标签，或每次上传时填写。
2
注册 GitHub 账号
1. 打开 github.com，点击「Sign up」
2. 填写用户名（建议简短，会出现在网址中）、邮箱、密码
3. 完成人机验证，进入邮箱点击验证链接
4. 登录 GitHub，完成注册
💡 用户名会出现在你的网址中，例如用户名 zhangsan → 网址为 zhangsan.github.io/research-lib
3
新建仓库
1. 登录 GitHub 后，点击右上角「+」→「New repository」
2. Repository name 填写：research-lib
3. 选择「Public」（公开，GitHub Pages 免费版需要公开）
4. 勾选「Add a README file」
5. 点击「Create repository」
4
上传项目文件
1. 进入刚创建的仓库页面
2. 点击「Add file」→「Upload files」
3. 将下载的 index.html 拖入上传区域
4. 在页面下方「Commit changes」区域直接点击绿色按钮提交
5. 上传完成后仓库里应该能看到 index.html
💡 下载文件：点击下方「⬇ 下载 index.html」按钮获取项目文件。
5
开启 GitHub Pages
1. 在仓库页面，点击顶部「Settings」（设置）
2. 左侧菜单找到「Pages」
3. 在「Source」下拉框选择「Deploy from a branch」
4. Branch 选择「main」，目录选「/ (root)」
5. 点击「Save」
6. 等待约 1-2 分钟，页面顶部会出现你的网址
💡 网址格式：https://你的用户名.github.io/research-lib
将此网址填入「设置」标签的「部署域名」，之后每次导出的 HTML 链接才能被外部访问。
6
发布研究报告（日常使用流程）
1. 打开你的资料库页面（已部署的 GitHub Pages 地址）
2. 在「上传报告」标签上传 PDF，AI 生成摘要
3. 点击「⬇ 导出 HTML 页面」，下载生成的报告页面文件（如 1716000000000.html）
4. 回到 GitHub 仓库，新建文件夹路径：result/2026/05/19/
   （点击「Add file」→「Create new file」，在文件名栏输入 result/2026/05/19/1716000000000.html）
5. 粘贴导出文件内容，提交
6. 约 1 分钟后，分享链接即可永久访问 ✓
💡 更简便的方法：每次直接把导出的 HTML 上传到仓库根目录，链接格式改为 https://用户名.github.io/research-lib/文件名.html，同样可以分享。
